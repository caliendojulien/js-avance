Promesse, await et async

Objectifs

L’objectif de cette démonstration est l’utilisation des promesses et des mots clés async et await.

Lancer le script javascript en mode debug pour avoir le bon affichage des émojis dans la console.

Chainage

Un des grands intérêts des promesses est le fait de pouvoir la chainer.

fetch('https://pokeapi.co/api/v2/pokemon/pikachu')
    .then(r => r.json())
    .then(reponse => {
        // console.log(reponse["sprites"]["other"]["home"]["front_female"]);
        let image = document.createElement('img');
        image.src = reponse["sprites"]["other"]["home"]["front_female"];
        document.getElementById('pikachu').appendChild(image);
    })

Ici,

  • fetch() retourne une promesse à la ligne 1

  • r.json() retourne une promesse à la ligne 2

Et les deux sont "chaînées".

Erreurs

N’importe quelle erreur dans la chaine de promesse peut être récupérée grâce à catch()

fetch('')
.catch (erreur => console.log(erreur));

Intercepter une erreur entraîne l’arrêt immédiat de toute la chaîne de promesses.

Créer une méthode asynchrone

Si on imagine une fonction qui incrémente un indice jusqu’à un très grand nombre, celle-ci prendra du temps à s’exécuter.

De plus, elle bloque le déroulé du programme puisque l’instruction suivante attend que la précédente soit terminée.

console.group('synchrone');

function synchrone() {
    let i = 0;
    while (i < 1000000000) {
        i++;
    }
}

console.time('ensemble');
console.time('avant');
console.log('avant');
console.timeEnd('avant');
console.time('synchrone');
synchrone();
console.log('synchrone')
console.timeEnd('synchrone');
console.time('apres');
console.log('apres');
console.timeEnd('apres');
console.timeEnd('ensemble');
console.groupEnd();

Lorsque les instructions situées après la méthode n’ont pas besoin qu’elle soit terminée, c’est une bonne occasion de la rendre asynchrone grâce aux promesses.

console.group('promesse');

function promesse() {
    return Promise.resolve()
        .then(
            t => {
                let i = 0;
                while (i < 1000000000) {
                    i++;
                }
                return ('Promesse');
            }
        );

}

console.time('ensemble');
console.time('avant');
console.log('avant');
console.timeEnd('avant');
console.time('promesse');
promesse().then((retour) => console.log(retour));
console.timeEnd('promesse');
console.time('apres');
console.log('apres');
console.timeEnd('apres');
console.timeEnd('ensemble');
console.groupEnd();

Les promesses peuvent être difficiles à suivre et à lire. Async et Await sont du sucre syntaxique permettant de clarifier l’utilisation des promesses.

Async / await

Async

async permet de simplifier l’écriture d’une méthode asynchrone passant de ça :

const getAnimal = (nom) => {
    const animaux = {
        'singe': '🐵',
        'chien': '🐶',
        'loup': '🐺',
        'renard': '🦊',
        'licorne': '🦄',
        'lapin': '🐰'
    }
    return Promise.resolve(animaux[nom]);
}
getAnimal('licorne').then(console.log);

a ca :

const getAnimalAsync = async (nom) => {
    const animaux = {
        'singe': '🐵',
        'chien': '🐶',
        'loup': '🐺',
        'renard': '🦊',
        'licorne': '🦄',
        'lapin': '🐰'
    }
    return animaux[nom];
}
getAnimalAsync('licorne').then(console.log);

Les deux méthodes retournent exactement le même résultat.

Await

A l’inverse, await permet d’attendre le retour d’un appel asynchrone avant de passer a l’instruction suivante.

const zoo1 = async () => {
    const lapin = await getAnimal('lapin');
    const loup = await getAnimal('loup');
    return [lapin, loup]
}
zoo1().then(console.log)

Ici, une énorme erreur est commise. Mais c’est pédagogique. C’est le cas typique d’un await mal utilisé.

Le deuxième appel asynchrone n’est effectué qu’après le premier. Pourtant, il aurait été préférable de lancer les deux appels en même temps et d’attendre le retour des deux dans le return.

Comme ici :

const zoo2 = async () => {
    const lapin = getAnimal('lapin');
    const loup = getAnimal('loup');
    const enclos = await Promise.all([lapin, loup]);
    return enclos;
}
zoo2().then(console.log)
Rappeler que l’important, quand on travaille en asynchrone, c’est de ne pas bloquer l’exécution du programme si ce n’est pas nécessaire.
